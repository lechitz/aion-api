# Database Infrastructure

**Path:** `infrastructure/db`

## Purpose

This folder owns versioned schema changes and deterministic local data bootstrap assets.

## Current Areas

| Area | Responsibility |
| --- | --- |
| `migrations/` | schema evolution and rollback-safe SQL |
| `seed/` | disposable local or QA data sets and helper seed assets |
| Workspace `../backups/aion-api/dev/` | local dump files generated by `make backup-dev` from the `aion-api` repo root; ignored by Git |

## Boundary Rules

- migrations own schema shape; seeds must not quietly compensate for missing schema work
- SQL here is operational code and should stay reviewable and deterministic

## Validate

```bash
make migrate-up
make seed-all
make backup-dev
```

## Risks And Compatibility Notes

- drift between migrations and seeds creates false confidence in local setups
- run `make backup-dev` before local reset, restore, destructive seed cleanup, or migration experiments that can affect real user-entered data
- by default, dev backups are stored outside the repo at `../backups/aion-api/dev/` from the `aion-api` root
- personal `my` backups are stored at `../backups/aion-api/my/` from the `aion-api` root
- restore is intentionally gated: use `make restore-dev CONFIRM_RESTORE=YES BACKUP_FILE=../backups/aion-api/dev/<file>.dump`
